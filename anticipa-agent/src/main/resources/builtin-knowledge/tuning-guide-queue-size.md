# 队列容量调参指南

## 基本公式

```
队列容量 = 预估 QPS × 响应时间 SLA（秒）
```

例如：
- QPS = 2000，响应时间 SLA = 1 秒
- 队列容量 = 2000 × 1 = 2000

## 考虑因素

### 流量波动
实际流量并非恒定，会随时间波动。建议在公式基础上额外增加缓冲：

```
最终容量 = 公式结果 × 1.5 ~ 2.0
```

### 线程池处理延迟
当线程数已满时，队列是唯一缓冲区。队列越长，任务的等待时间越长。

### 业务容忍度
- **实时交互**（Web 请求）：队列不宜过大，1000~5000
- **异步处理**（MQ 消费）：队列可以大一些，5000~50000
- **定时/批处理**：队列可以很大，取决于内存上限

## 典型场景参考

| 场景 | QPS | SLA | 建议队列容量 |
|---|---|---|---|
| API 网关 | 10000 | 200ms | 2000~4000 |
| 订单处理 | 500 | 1s | 500~1000 |
| 日志处理 | 50000 | 5s | 25000~50000 |
| 消息推送 | 2000 | 2s | 4000~8000 |

## 调整步骤
1. 按公式计算基准值
2. 观察实际队列深度曲线（峰值和均值）
3. 如果队列一直为空 → 适当减小队列容量
4. 如果队列经常满 → 检查是否需要扩容或增加线程数
5. 内存允许范围内，队列容量可适当增加以应对突发流量
