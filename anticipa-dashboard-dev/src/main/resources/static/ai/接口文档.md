## 二、登录认证

### 2.1 登录接口

**接口地址**: `POST /api/anticipa-dashboard/auth/login`

**请求参数**:

```json
{
  "username": "用户名",
  "password": "密码"
}
```

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "用户ID",
    "accessToken": "访问令牌",
    "realName": "真实姓名",
    "username": "用户名"
  }
}
```

**默认测试账号**:

| 用户名 | 密码 |
|--------|------|
| admin | admin |
| test | test |
| wjq | wjq |

### 2.2 登录状态存储

登录成功后，前端需要将 `accessToken` 存储到 `localStorage`：

```javascript
// 存储
localStorage.setItem('accessToken', data.accessToken);
localStorage.setItem('userInfo', JSON.stringify({
    id: data.id,
    username: data.username,
    realName: data.realName
}));

// 检查登录状态
function checkLoginStatus() {
    const token = localStorage.getItem('accessToken');
    if (!token) {
        window.location.href = 'ai-login.html';
        return false;
    }
    return true;
}

// 退出登录
function logout() {
    localStorage.removeItem('accessToken');
    localStorage.removeItem('userInfo');
    window.location.href = 'ai-login.html';
}
```

## 四、接口详情

### 4.1 发送聊天消息

**接口地址**: `POST /api/ai/chat`

**请求参数**:

```json
{
  "sessionId": "可选，会话ID，不传自动生成新的",
  "message": "必填，用户消息",
  "threadPoolId": "可选，指定线程池ID"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sessionId | String | 否 | 会话ID，用于保持对话上下文。不传则自动生成 |
| message | String | 是 | 用户发送的消息内容 |
| threadPoolId | String | 否 | 指定要查询的线程池ID |

**请求示例**:

```bash
curl -X POST http://localhost:9010/api/ai/chat \
  -H "Content-Type: application/json" \
  -d '{
    "sessionId": "abc123",
    "message": "查看所有线程池状态"
  }'
```

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "sessionId": "abc123",
    "content": "**线程池状态：order-service**\n\n| 指标 | 数值 |\n|------|------|\n| 核心线程数 | 10 |",
    "timestamp": 1712908800000,
    "success": true,
    "intent": "QUERY_STATUS",
    "suggestions": ["查看队列使用情况", "分析线程池性能", "获取优化建议"]
  }
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200表示成功 |
| message | String | 响应消息 |
| data.sessionId | String | 会话ID |
| data.content | String | AI回复内容（支持Markdown格式） |
| data.timestamp | Long | 响应时间戳 |
| data.success | Boolean | 是否成功 |
| data.intent | String | 识别到的用户意图 |
| data.suggestions | List<String> | 建议的追问内容 |

### 3.2 获取对话历史

**接口地址**: `GET /api/ai/history`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sessionId | String | 是 | 会话ID |
| limit | Integer | 否 | 限制返回条数，默认50 |

**请求示例**:

```bash
curl "http://localhost:9010/api/ai/history?sessionId=abc123&limit=10"
```

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": "msg-001",
      "sessionId": "abc123",
      "role": "user",
      "content": "查看所有线程池状态",
      "timestamp": 1712908800000,
      "messageType": "TEXT"
    },
    {
      "id": "msg-002",
      "sessionId": "abc123",
      "role": "assistant",
      "content": "以下是所有线程池的状态...",
      "timestamp": 1712908801000,
      "messageType": "TEXT"
    }
  ]
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 消息唯一ID |
| sessionId | String | 会话ID |
| role | String | 角色：user（用户）/ assistant（AI助手） |
| content | String | 消息内容 |
| timestamp | Long | 消息时间戳 |
| messageType | String | 消息类型：TEXT/MARKDOWN/CODE/TABLE/IMAGE |

### 3.8 流式聊天（可选）

**接口地址**: `GET /api/ai/chat/stream`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sessionId | String | 否 | 会话ID |
| message | String | 是 | 消息内容 |
| threadPoolId | String | 否 | 线程池ID |

**响应格式**: Server-Sent Events (SSE)

**响应示例**:

```
data: {"chunk":"这是","token":"0"}

data: {"chunk":"AI的","token":"1"}

data: {"chunk":"回复","token":"2"}

data: {"chunk":"内容","token":"3"}

data: [DONE]

data: {"sessionId":"abc123","success":true,"intent":"QUERY_STATUS"}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| chunk | String | 文本片段 |
| token | String | 片段序号 |
| [DONE] | - | 流结束标记 |
| sessionId | String | 会话ID（最后一条） |
| success | Boolean | 是否成功（最后一条） |
| intent | String | 识别意图（最后一条） |

## 六、前端对接示例

### 5.1 使用 Fetch API

```javascript
// 发送消息
async function sendMessage(sessionId, message) {
    try {
        const response = await fetch('/api/ai/chat', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify({
                sessionId: sessionId,
                message: message
            })
        });
        const result = await response.json();
        if (result.code === 200) {
            return result.data;
        } else {
            throw new Error(result.message);
        }
    } catch (error) {
        console.error('发送消息失败:', error);
        throw error;
    }
}

// 获取对话历史
async function getHistory(sessionId, limit = 50) {
    try {
        const response = await fetch(
            `/api/ai/history?sessionId=${sessionId}&limit=${limit}`
        );
        const result = await response.json();
        if (result.code === 200) {
            return result.data;
        } else {
            throw new Error(result.message);
        }
    } catch (error) {
        console.error('获取历史失败:', error);
        throw error;
    }
}

// 清空对话历史
async function clearHistory(sessionId) {
    try {
        const response = await fetch(
            `/api/ai/history?sessionId=${sessionId}`,
            { method: 'DELETE' }
        );
        const result = await response.json();
        return result.code === 200;
    } catch (error) {
        console.error('清空历史失败:', error);
        throw error;
    }
}

// 检查服务状态
async function checkStatus() {
    try {
        const response = await fetch('/api/ai/status');
        const result = await response.json();
        if (result.code === 200) {
            return result.data;
        }
        return null;
    } catch (error) {
        console.error('检查状态失败:', error);
        return null;
    }
}
```

### 5.2 使用 Axios

```javascript
import axios from 'axios';

// 创建axios实例
const aiApi = axios.create({
    baseURL: '/api/ai',
    timeout: 30000
});

// 发送消息
const sendMessage = async (sessionId, message) => {
    try {
        const { data } = await aiApi.post('/chat', {
            sessionId,
            message
        });
        if (data.code === 200) {
            return data.data;
        }
        throw new Error(data.message);
    } catch (error) {
        console.error('发送消息失败:', error);
        throw error;
    }
};

// 获取历史
const getHistory = async (sessionId, limit = 50) => {
    try {
        const { data } = await aiApi.get('/history', {
            params: { sessionId, limit }
        });
        if (data.code === 200) {
            return data.data;
        }
        throw new Error(data.message);
    } catch (error) {
        console.error('获取历史失败:', error);
        throw error;
    }
};

// 获取示例问题
const getExamples = async () => {
    try {
        const { data } = await aiApi.get('/examples');
        if (data.code === 200) {
            return data.data;
        }
        return [];
    } catch (error) {
        console.error('获取示例失败:', error);
        return [];
    }
};
```

### 5.3 Vue 3 组合式 API 示例

```javascript
// useAI.js
import { ref } from 'vue';

export function useAI() {
    const messages = ref([]);
    const sessionId = ref(null);
    const isLoading = ref(false);

    // 发送消息
    const sendMessage = async (message) => {
        isLoading.value = true;
        try {
            const response = await fetch('/api/ai/chat', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify({
                    sessionId: sessionId.value,
                    message
                })
            });
            const result = await response.json();

            if (result.code === 200) {
                // 更新会话ID
                if (!sessionId.value && result.data.sessionId) {
                    sessionId.value = result.data.sessionId;
                }
                return result.data;
            }
            throw new Error(result.message);
        } finally {
            isLoading.value = false;
        }
    };

    // 获取历史
    const loadHistory = async () => {
        if (!sessionId.value) return;

        const response = await fetch(
            `/api/ai/history?sessionId=${sessionId.value}`
        );
        const result = await response.json();

        if (result.code === 200) {
            messages.value = result.data;
        }
    };

    // 清空对话
    const clearChat = async () => {
        if (!sessionId.value) return;

        await fetch(`/api/ai/history?sessionId=${sessionId.value}`, {
            method: 'DELETE'
        });
        messages.value = [];
    };

    return {
        messages,
        sessionId,
        isLoading,
        sendMessage,
        loadHistory,
        clearChat
    };
}
```

## 七、注意事项

### 7.1 启用 AI 功能

确保在 `application.yaml` 中正确配置：

```yaml
anticipa:
  ai:
    enable: true
    openai:
      api-key: your-api-key
      base-url: https://your-proxy.com/v1
      model: gpt-3.5-turbo
```

### 7.2 CORS 跨域配置

如果前后端分离部署，需确保后端配置了跨域支持。

### 7.3 Markdown 渲染

AI 返回的 `content` 字段支持 Markdown 格式，建议使用 `marked.js` 或类似库进行渲染：

```html
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
<script>
    const htmlContent = marked.parse(markdownContent);
    document.getElementById('chat-content').innerHTML = htmlContent;
</script>
```

### 7.4 会话管理

- 首次调用 `chat` 接口时，`sessionId` 传空或不传，后端会自动生成
- 后续对话需携带返回的 `sessionId` 以保持上下文
- 可调用 `clearHistory` 清空指定会话的历史记录
